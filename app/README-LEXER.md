README — Демонстрація лексера (RD tokenizer)
=============================================

Коротко
-------
Цей файл пояснює, як швидко запустити демон лексера, що використовує реалізацію рекурсивного спуску (RD tokenizer) у проекті.

Де знаходиться код
------------------
- Демон: `org.example.lexer.LexerDemo`
- Реалізація токенайзера: `org.example.parser.RecursiveDescentParser`

Що робить демо
---------------
Демон зчитує текстовий файл з кодом Sigma і виводить дамп токенів на стандартний вивід (термінал). Це зручно для показу роботи лексера при захисті лабораторної роботи.

Запуск (рекомендований)
----------------------
1) Через Gradle-task (зроблено у `app/build.gradle`):

```powershell
.\gradlew.bat :app:runLexerDemo
```

За замовчуванням цей таск використовує файл `syntax_error_tests.sigma` у корені репозиторію.

Запуск з іншим файлом
---------------------
1) Легко: відкрити проект у вашому IDE (IntelliJ/VS Code) і запустити `org.example.lexer.LexerDemo` як Java-програму, передавши аргумент — шлях до потрібного `.sigma` файлу.

2) Або зібрати jar і запустити клас напряму (приклад):

```powershell
.\gradlew.bat :app:jar
# потім (підлаштуйте ім'я jar, якщо інше):
# java -cp app\build\libs\<your-jar>.jar org.example.lexer.LexerDemo path\to\file.sigma
```

Примітки для здачі
-------------------
- Для демонстрації під час захисту просто запустіть команду `:app:runLexerDemo` — це виведе токени у термінал.
- Демон показує вихід роботи RD-лексера (не ANTLR). Це вмотивовано тим, що ми реалізували власний токенайзер для лабораторної роботи.

Питання/покращення
------------------
- Якщо хочете, я можу додати опцію CLI (`--input <file>`) до таску `runLexerDemo`, щоб можна було передавати інший файл з командного рядка без збірки/IDE. Напишіть, якщо потрібно.
